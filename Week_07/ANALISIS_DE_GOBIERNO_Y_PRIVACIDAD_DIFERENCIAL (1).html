##Contexto del Problema: Analítica Gubernamental y Asignación Ética de Subsidios

El Ministerio de Asignación de Subsidios (entidad gubernamental ficticia) administra programas de asistencia financiera para poblaciones vulnerables. Para optimizar la distribución de recursos y detectar patrones de necesidad, la dirección de analítica de datos requiere ejecutar modelos de Machine Learning (Clustering) sobre la base de datos de solicitantes.

Sin embargo, los registros contienen Información de Identificación Personal (PII) y datos financieros altamente sensibles, como el ingreso mensual estimado y el puntaje de vulnerabilidad social. Exponer estos datos exactos viola el principio de confidencialidad estipulado en las leyes de protección de datos (ej. Habeas Data, GDPR). El objetivo de este proyecto es inyectar ruido estadísticamente calibrado (Privacidad Diferencial) a las variables sensibles para crear un "dataset protegido" que garantice el anonimato de los ciudadanos, pero que conserve la utilidad analítica suficiente para agrupar a los individuos según sus necesidades socioeconómicas

¿Qué se observó?: Se corrigieron los errores tipográficos, se imputaron los valores nulos usando la mediana poblacional y se transformaron las variables categóricas (Ciudad, Programa) mediante One-Hot Encoding*. Además, se eliminaron los IDs y las fechas.
¿Qué significa?: Se estandarizó la información. El One-Hot Encoding* convirtió el texto en matrices binarias (ceros y unos), y la eliminación del ID asegura que el modelo no busque patrones en un identificador aleatorio.
* ¿Cómo impacta el modelo?: Es el paso más crítico para DBSCAN. Al ser un modelo basado en densidad, calcula distancias matemáticas (Euclidianas). Si se hubieran dejado datos tipo texto (object), el modelo habría arrojado un error fatal. Ahora los datos son 100% compatibles matemáticamente.

def aplicar_ruido_laplace(serie, epsilon, sensibilidad):
    escala = sensibilidad / epsilon
    ruido = np.random.laplace(loc=0.0, scale=escala, size=len(serie))
    return np.maximum(0, serie + ruido)

df_ml_protegido = df_ml.copy()

# Parámetros de Privacidad
epsilon_ingreso = 0.5
sensibilidad_ingreso = 2000000

epsilon_puntaje = 1.0
sensibilidad_puntaje = 100

# Inyección de ruido en variables sensibles
df_ml_protegido['Ingreso_Mensual'] = aplicar_ruido_laplace(
    df_ml['Ingreso_Mensual'], epsilon_ingreso, sensibilidad_ingreso
)
df_ml_protegido['Puntaje_Vulnerabilidad'] = aplicar_ruido_laplace(
    df_ml['Puntaje_Vulnerabilidad'], epsilon_puntaje, sensibilidad_puntaje
)

print("Comparación de Medias (Original vs Protegido):")

📝 Análisis de Privacidad Diferencial


* ¿Qué se observó?: Al aplicar el Mecanismo de Laplace a los Ingresos_Mensuales y el Puntaje_Vulnerabilidad, la media de los ingresos aumentó drásticamente en comparación con el dataset original.
* ¿Qué significa?: El algoritmo inyectó valores aleatorios masivos basados en una sensibilidad de 2 millones y un presupuesto de privacidad estricto ($\epsilon = 0.5$). Como se aplicó una restricción para evitar ingresos negativos (reemplazándolos por cero), el ruido positivo no tuvo contrapeso negativo, inflando el promedio.
* ¿Cómo impacta el modelo?: La exactitud a nivel de registro desaparece. La información individual está ahora protegida contra reidentificación, pero la varianza de los datos aumentó artificialmente, lo que alterará las densidades que el modelo intente encontrar.

def metricas_dbscan(labels, nombre):
    n_clusters = len(set(labels)) - (1 if -1 in labels else 0)
    ruido = list(labels).count(-1) / len(labels) * 100
    print(f"--- {nombre} ---")
    print(f"Clústeres formados: {n_clusters}")
    print(f"Puntos anómalos (Ruido): {ruido:.2f}%\n")

📝 Análisis de Resultados DBSCAN


* ¿Qué se observó?: Utilizando los mismos hiperparámetros (eps=1.5, min_samples=10), el modelo original agrupó la mayoría de los datos, mientras que el modelo protegido clasificó un porcentaje muchísimo mayor de ciudadanos como "ruido" o valores anómalos (-1).
* ¿Qué significa?: El ruido inyectado por SmartNoise "empujó" a los ciudadanos fuera de sus grupos de subsidios originales. Los clústeres densos perdieron su cohesión porque las distancias entre los puntos se ampliaron artificialmente.
* ¿Cómo impacta el modelo?: Demuestra que no se puede simplemente "conectar" un dataset protegido a un modelo pre-entrenado. Si se quiere mantener el modelo en producción con datos anonimizados, se deben recalibrar los hiperparámetros de DBSCAN (aumentar el radio eps) para tolerar la nueva dispersión estadística.

PARTE 7: INTERPRETACIÓN DE RESULTADOS


==========================================


1. ¿El ruido destruyó completamente los patrones?
No, pero alteró significativamente la densidad local. La visualización t-SNE demuestra que las macro-agrupaciones (ciudadanos con características similares) persisten, pero la dispersión interna aumentó, diluyendo las fronteras estrictas de los clústeres.

2. ¿Qué tanto cambió el clustering?
El cambio fue severo bajo la métrica de asignación de DBSCAN. Debido al ruido diferencial, la distancia euclidiana entre ciudadanos aumentó, provocando que muchos puntos que originalmente eran "núcleo" fueran empujados fuera del umbral (eps), siendo reclasificados como ruido/outliers (-1).

3. ¿Qué se perdió y qué se protegió?
* Se perdió: La exactitud a nivel de registro [Individual Accuracy]. Es imposible detectar micro-anomalías o fraudes específicos con el mismo modelo sin recalibrar los hiperparámetros.
Se protegió: El anonimato y la confidencialidad financiera frente a ataques de vinculación (linkage attacks*). Hay negabilidad plausible sobre el ingreso y vulnerabilidad de cualquier ciudadano.

4. ¿Vale la pena sacrificar precisión por privacidad?
Totalmente. En el sector público, el daño legal (Habeas Data) y ético de una filtración de PII supera la pérdida de precisión en un modelo no supervisado.

PARTE 8: RELACIÓN CON MACHINE LEARNING REAL


==========================================


En entornos de producción gubernamental o Fintech, el cruce de datos para asignar subsidios está estrictamente vigilado por leyes como la Ley 1581 de 2012 (Habeas Data) en Colombia o el GDPR. Utilizar Privacidad Diferencial (SmartNoise) permite que los equipos de ciencia de datos modelen patrones de pobreza o detecten fraudes macroeconómicos (clustering) sin exponerse a demandas legales. Se implementa Privacy by Design: el modelo opera sobre representaciones estadísticas seguras, y solo los auditores autorizados verifican los datos en crudo en etapas de control final.
